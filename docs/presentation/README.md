# 発表資料テンプレート

ハンズオン終了後の発表で使用する資料テンプレートです。

---

## 📋 使い方

### 1. テンプレートをコピー

`template.md` をコピーして、自分の名前で保存してください。

例：
```bash
cp template.md 発表資料_山田太郎.md
```

### 2. 各セクションを記入

テンプレートの各セクションに、自分の開発プロセスや成果物について記入してください。

### 3. Cursorで作成してもOK

この資料自体もCursorに作成してもらって構いません！

例：
```
発表資料を作成してください。以下の内容を含めてください：
- 成果物の概要
- 開発プロセス
- 使用したプロンプトの例
- つまずいた点と解決方法
- 感想・学び
```

---

## 📝 記入のポイント

### 成果物の概要

- どんなECサイトを作ったか
- 実装した機能を明確に

### 開発プロセス

- どんな順序で開発したか
- 効果的だったプロンプトを2〜3個紹介
- 特に工夫した点を3つまで

### つまずいた点と解決方法

- どんな問題に遭遇したか
- どう解決したか（プロンプトでの解決方法）
- 学びや気づき

### 感想・学び

- AIエージェント開発の感想
- API連携で驚いたこと
- 今後やってみたいこと

---

## ⏰ 発表時間

- **発表**: 7分
- **フィードバック**: 3分
- **合計**: 10分

### 時間配分の目安

- 成果物概要: 1分
- 開発プロセス: 4分
- デモ: 3分
- 感想・学び: 1分

---

## 💡 発表のコツ

### 話し方のポイント

- **結論ファースト**: 最初に「何を作ったか」を端的に説明
- **具体例を使う**: プロンプト例、エラー例など具体的に
- **失敗も共有**: つまずいた点も価値ある学び
- **API連携の驚き**: バイブコーディングでAPI連携できた感動を伝える
- **ポジティブに**: AIとの協働開発を楽しむ姿勢

### デモのポイント

- **ゆっくり操作**: 視聴者が理解できるペースで
- **声に出す**: 何をしているか実況する
- **APIを意識**: 「このデータはAPIから取得しています」と説明
- **トラブル対応**: 動かなくても慌てない、その対応も学び
