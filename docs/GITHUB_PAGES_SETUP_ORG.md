# GitHub Pages設定（組織制限がある場合）

## ⚠️ 現在の状況

組織（`aainc`）の管理者がGitHub Pagesの作成を無効にしているため、API経由での設定ができません。

## 🔧 解決方法

### 方法1: 組織の管理者に依頼（推奨）

組織の管理者に以下を依頼してください：

1. **組織のSettings** > **Pages** に移動
2. **「Allow members to publish sites」** を有効化
3. または、特定のリポジトリのみ許可する設定を確認

### 方法2: 手動で設定

組織の設定が変更できない場合でも、手動で設定できる場合があります：

1. リポジトリの **Settings** > **Pages** に移動
2. 手動で設定を試みる
3. エラーが表示される場合は、組織の管理者に依頼が必要です

### 方法3: 組織の設定を確認

組織の管理者は以下で設定を確認できます：

1. **組織のSettings** > **Pages**
2. **「Allow members to publish sites」** の設定を確認
3. 必要に応じて有効化

---

## 📝 補足

組織レベルの制限がある場合、以下のいずれかが必要です：

- 組織の管理者権限
- 組織の管理者による設定変更の依頼
